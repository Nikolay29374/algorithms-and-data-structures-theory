## 1. Алгоритмы

**Алгоритм** — это чёткая последовательность действий, которую необходимо выполнить для решения определённой задачи.

Алгоритмы используются почти во всех областях информатики: программировании, математике, обработке данных, искусственном интеллекте, базах данных, сетях и других направлениях.

Примеры алгоритмов из повседневной жизни:

- рецепт приготовления блюда;
- инструкция по сборке мебели;
- маршрут от одной точки до другой;
- последовательность действий при оплате покупки.

### Основные свойства алгоритма

1. **Точность**  
   Каждый шаг алгоритма должен быть чётко определён и пониматься однозначно.

2. **Дискретность**  
   Алгоритм должен состоять из отдельных последовательных шагов.

3. **Детерминированность**  
   При одинаковых входных данных алгоритм должен выдавать один и тот же результат.

4. **Конечность**  
   Алгоритм должен завершиться после конечного числа шагов.

5. **Массовость**  
   Алгоритм должен быть применим к широкому классу однотипных задач.

6. **Результативность**  
   После выполнения алгоритма должен быть получен результат, удовлетворяющий условиям задачи.

7. **Понятность**  
   Каждый шаг алгоритма должен быть понятен исполнителю.

### Сложность алгоритма

**Сложность алгоритма** — это количество ресурсов, которые использует алгоритм при работе.

Обычно рассматривают два основных вида сложности:

- **временная сложность** — сколько времени или элементарных операций требуется алгоритму;
- **пространственная сложность** — сколько дополнительной памяти требуется алгоритму.

Сложность обычно выражают как функцию от размера входных данных `n`.

Например, если при увеличении размера входных данных в 2 раза время работы алгоритма тоже увеличивается примерно в 2 раза, то такой алгоритм имеет линейную сложность `O(n)`.

## 2. Модель RAM (Random Access Machine)

**RAM-модель** — это теоретическая модель вычислений, которая используется для анализа сложности алгоритмов.

В этой модели считается, что каждая элементарная операция выполняется за фиксированное и одинаковое количество времени.

### Элементарные операции в RAM-модели

К элементарным операциям обычно относят:

- арифметические операции: сложение, вычитание, умножение, деление, взятие остатка;
- операции сравнения: `<`, `>`, `<=`, `>=`, `==`, `!=`;
- операции присваивания;
- операции доступа к данным;
- базовые логические операции: `AND`, `OR`, `NOT`.

### Важные особенности RAM-модели

- память модели считается неограниченной;
- инструкции выполняются последовательно, одна за другой;
- каждая элементарная операция занимает `O(1)` времени;
- детали конкретного процессора, языка программирования и реализации обычно не учитываются.

RAM-модель позволяет сосредоточиться не на технических особенностях компьютера, а на общей структуре алгоритма.

### Пример подсчёта операций

```python
import sys

min_value = sys.maxsize
max_value = -sys.maxsize - 1

for x in array:
    if x < min_value:
        min_value = x
    if x > max_value:
        max_value = x
```

Этот код ищет минимальное и максимальное значение в массиве `array`.

Подсчитаем количество элементарных операций:

1. Присваивание `min_value` и `max_value` — 2 операции.
2. Получение значений `sys.maxsize` и `-sys.maxsize - 1` — несколько константных операций.
3. Цикл `for` проходит по всем `n` элементам массива.
4. Для каждого элемента выполняются сравнения:
   - `x < min_value`;
   - `x > max_value`.
5. Если условие выполняется, происходит присваивание нового значения.

В худшем случае внутри цикла выполняется константное количество операций для каждого элемента. Поэтому сложность алгоритма:

```text
T(n) = O(n)
```

## 3. Асимптотическая сложность

**Асимптотическая сложность** показывает, как растёт время работы алгоритма или объём используемой памяти при увеличении размера входных данных.

Главная идея асимптотики — смотреть не на точное количество операций, а на скорость роста функции при больших `n`.

### Основные асимптотические обозначения

| Обозначение | Название | Что описывает |
|---|---|---|
| `O(f(n))` | Big O | Верхнюю границу роста |
| `Ω(f(n))` | Big Omega | Нижнюю границу роста |
| `Θ(f(n))` | Big Theta | Точную асимптотическую оценку |

### Big O

`O(f(n))` описывает верхнюю границу сложности алгоритма.

Говорят, что:

```text
T(n) = O(f(n))
```

если существуют константы `c` и `n0`, такие что для всех `n ≥ n0` выполняется:

```text
0 ≤ T(n) ≤ c · f(n)
```

Big O часто используют для оценки худшего случая.

Пример: сортировка пузырьком.

```python
def bubble_sort(array):
    n = len(array)
    for i in range(n):
        for j in range(0, n - i - 1):
            if array[j] > array[j + 1]:
                array[j], array[j + 1] = array[j + 1], array[j]
    return array
```

В худшем случае выполняются два вложенных цикла, поэтому временная сложность:

```text
O(n²)
```

### Big Omega

`Ω(f(n))` описывает нижнюю границу сложности алгоритма.

Говорят, что:

```text
T(n) = Ω(f(n))
```

если существуют константы `c` и `n0`, такие что для всех `n ≥ n0` выполняется:

```text
0 ≤ c · f(n) ≤ T(n)
```

Пример: линейный поиск.

```python
def linear_search(array, element):
    for i in range(len(array)):
        if array[i] == element:
            return i
    return -1
```

Лучший случай — искомый элемент находится в начале массива. Тогда сложность `Ω(1)`.  
В среднем и худшем случаях сложность будет `O(n)`.

### Big Theta

`Θ(f(n))` описывает точную асимптотическую границу, когда верхняя и нижняя оценки совпадают.

Говорят, что:

```text
T(n) = Θ(f(n))
```

если существуют константы `c1`, `c2` и `n0`, такие что для всех `n ≥ n0` выполняется:

```text
0 ≤ c1 · f(n) ≤ T(n) ≤ c2 · f(n)
```

Пример: вычисление факториала.

```python
def factorial(n):
    if n == 0:
        return 1
    return n * factorial(n - 1)
```

Функция делает `n` рекурсивных вызовов, поэтому:

```text
T(n) = Θ(n)
```

### Лучший, средний и худший случаи

| Случай | Обозначение | Описание |
|---|---|---|
| Best Case | Лучший случай | Минимальное количество операций |
| Average Case | Средний случай | Ожидаемое количество операций |
| Worst Case | Худший случай | Максимальное количество операций |

### Частые классы сложности

| Сложность | Название | Пример |
|---|---|---|
| `O(1)` | Константная | Доступ к элементу массива по индексу |
| `O(log n)` | Логарифмическая | Бинарный поиск |
| `O(n)` | Линейная | Линейный поиск |
| `O(n log n)` | Линеаритмическая | Быстрая сортировка, сортировка слиянием |
| `O(n²)` | Квадратичная | Два вложенных цикла, bubble sort |
| `O(2ⁿ)` | Экспоненциальная | Наивное вычисление чисел Фибоначчи |

### Почему основание логарифма часто не указывают

В асимптотическом анализе основание логарифма обычно не важно, потому что логарифмы с разными основаниями отличаются только на константу.

```text
log_a(n) = log_b(n) / log_b(a)
```

Так как `1 / log_b(a)` — это константа, то:

```text
O(log_a n) = O(log_b n)
```

Поэтому часто пишут просто `O(log n)`.

### Соотношения между простейшими функциями

При больших `n` функции растут примерно в таком порядке:

```text
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ)
```

Логарифмические алгоритмы обычно эффективнее линейных, линейные — эффективнее квадратичных, а полиномиальные — эффективнее экспоненциальных.

## 4. Рекурсия

**Рекурсивная функция** — это функция, которая вызывает саму себя.

Рекурсия часто используется для решения задач, которые можно разбить на более простые подзадачи того же типа.

### Из чего состоит рекурсия

1. **Базовый случай**  
   Условие, при котором функция перестаёт вызывать саму себя.

2. **Рекурсивный шаг**  
   Вызов функции с новыми аргументами, которые приближают задачу к базовому случаю.

### Пример: факториал

```python
def factorial(n):
    if n == 0:
        return 1
    return n * factorial(n - 1)

result = factorial(5)
print(result)  # 120
```

В этом примере:

- базовый случай: `n == 0`;
- рекурсивный шаг: `factorial(n - 1)`.

### Достоинства рекурсии

- код может быть проще и понятнее;
- удобно описывать задачи, которые естественно имеют рекурсивную структуру;
- подходит для деревьев, графов, перебора вариантов, divide and conquer.

### Недостатки рекурсии

- возможное переполнение стека вызовов;
- дополнительные расходы памяти на стек;
- рекурсивный код иногда сложнее отлаживать;
- при неправильном базовом случае можно получить бесконечную рекурсию.

## 5. Оценка сложности рекурсивных алгоритмов

Оценивать рекурсивные алгоритмы сложнее, чем итеративные, потому что нужно учитывать не только работу одного вызова, но и количество рекурсивных вызовов.

Основные подходы:

1. **Рекуррентные соотношения**  
   Описывают время работы функции через время работы меньших подзадач.

2. **Дерево рекурсии**  
   Позволяет увидеть, сколько вызовов создаётся на каждом уровне.

3. **Математическая индукция**  
   Используется для доказательства оценки сложности.

4. **Мастер-теорема**  
   Позволяет быстро оценивать некоторые типы рекуррентных соотношений.

### Пример 1: факториал

```python
def factorial(n):
    if n == 0:
        return 1
    return n * factorial(n - 1)
```

Рекуррентное соотношение:

```text
T(n) = T(n - 1) + O(1)
```

Каждый вызов уменьшает `n` на 1, поэтому всего будет `n` вызовов.

```text
T(n) = O(n)
```

### Пример 2: наивное вычисление Фибоначчи

```python
def fibonacci(n):
    if n <= 1:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)
```

Рекуррентное соотношение:

```text
T(n) = T(n - 1) + T(n - 2) + O(1)
```

Каждый вызов порождает два новых вызова, поэтому строится большое дерево рекурсии.

```text
T(n) = O(2ⁿ)
```

### Пример 3: бинарный поиск

```python
def binary_search(arr, low, high, key):
    if high >= low:
        mid = (low + high) // 2

        if arr[mid] == key:
            return mid
        elif arr[mid] > key:
            return binary_search(arr, low, mid - 1, key)
        else:
            return binary_search(arr, mid + 1, high, key)
    else:
        return -1
```

Рекуррентное соотношение:

```text
T(n) = T(n / 2) + O(1)
```

На каждом шаге размер задачи уменьшается в 2 раза.

```text
T(n) = O(log n)
```

## 6. Мастер-теорема

**Мастер-теорема** применяется для оценки рекуррентных соотношений вида:

```text
T(n) = aT(n / b) + O(nᶜ)
```

где:

- `a` — количество подзадач;
- `n / b` — размер каждой подзадачи;
- `O(nᶜ)` — работа вне рекурсивных вызовов.

### Идея применения

Нужно сравнить работу рекурсивных вызовов с работой, которая выполняется на каждом уровне рекурсии.

Часто результат можно определить через сравнение `c` и `log_b(a)`.

### Пример 1

```text
T(n) = 2T(n / 2) + n log n
```

Здесь дополнительная работа больше, чем обычная линейная работа `n`, поэтому простая форма мастер-теоремы может быть недостаточно удобной для точной оценки. Однако можно ограничить функцию сверху и получить приближённую оценку.

### Пример 2

```text
T(n) = 2T(√n) + log₂ n
```

Иногда рекурренту удобно преобразовать заменой переменной.

Пусть:

```text
P(x) = T(2ˣ)
```

Тогда исходное соотношение можно привести к более привычному виду:

```text
P(x) = 2P(x / 2) + x
```

После применения мастер-теоремы получаем:

```text
T(n) = O(log n · log log n)
```

## 7. Структуры данных

**Структура данных** — это способ организации, хранения и управления данными, который позволяет эффективно использовать эти данные в алгоритмах.

Правильный выбор структуры данных может существенно повысить производительность программы.

### Основные структуры данных

#### 1. Массивы (Arrays)

**Массив** — это набор элементов фиксированного размера, которые хранятся в непрерывной области памяти.

Особенности:

- быстрый доступ к элементу по индексу: `O(1)`;
- размер часто задаётся заранее;
- вставка и удаление в середине могут быть дорогими: `O(n)`.

Примеры использования:

- хранение списков данных;
- векторы и матрицы;
- таблицы значений.

#### 2. Связные списки (Linked Lists)

**Связный список** — это структура данных, состоящая из узлов. Каждый узел хранит значение и ссылку на следующий узел.

Виды связных списков:

- односвязный список;
- двусвязный список;
- кольцевой список.

Примеры использования:

- динамическое управление памятью;
- реализация очередей и стеков;
- музыкальные плейлисты.

#### 3. Стеки (Stacks)

**Стек** — это структура данных, работающая по принципу **LIFO**: last in, first out — последним пришёл, первым вышел.

Основные операции:

- `push` — добавить элемент;
- `pop` — удалить верхний элемент;
- `peek` / `top` — посмотреть верхний элемент.

Примеры использования:

- стек вызовов функций;
- обработка скобочных последовательностей;
- обход деревьев и графов;
- отмена действий в программах.

#### 4. Очереди (Queues)

**Очередь** — это структура данных, работающая по принципу **FIFO**: first in, first out — первым пришёл, первым вышел.

Основные операции:

- `enqueue` — добавить элемент в конец очереди;
- `dequeue` — удалить элемент из начала очереди.

Примеры использования:

- планировщики задач;
- обработка запросов на сервере;
- обход графа в ширину `BFS`.

#### 5. Деревья (Trees)

**Дерево** — это иерархическая структура данных, состоящая из узлов.

Основные понятия:

- корень;
- родитель;
- потомок;
- лист;
- глубина;
- высота.

Примеры использования:

- файловые системы;
- индексы в базах данных;
- деревья решений;
- синтаксические деревья.

#### 6. Графы (Graphs)

**Граф** — это структура данных, состоящая из вершин и рёбер.

Графы бывают:

- ориентированные и неориентированные;
- взвешенные и невзвешенные;
- связные и несвязные.

Примеры использования:

- социальные сети;
- карты и маршруты;
- сети дорог;
- зависимости между объектами.

#### 7. Хеш-таблицы (Hash Tables)

**Хеш-таблица** — это структура данных, которая использует хеш-функцию для быстрого доступа к данным по ключу.

Особенности:

- средний доступ, вставка и удаление: `O(1)`;
- возможны коллизии;
- эффективность зависит от хеш-функции и коэффициента заполнения.

Примеры использования:

- словари;
- множества;
- кэширование;
- быстрый поиск данных.

### Зачем нужны структуры данных

Структуры данных помогают:

- эффективнее управлять памятью;
- ускорять алгоритмы;
- упрощать код;
- решать сложные задачи;
- абстрагироваться от низкоуровневых деталей реализации.

### План распределения задач

| Тема | Куда добавлять задачи |
|---|---|
| Алгоритмы | `tasks/algorithms/` |
| Подсчёт операций и RAM-модель | `tasks/ram-model/` |
| Асимптотическая сложность | `tasks/complexity/` |
| Рекурсия | `tasks/recursion/` |
| Рекуррентные соотношения и мастер-теорема | `tasks/master-theorem/` |
| Структуры данных | `tasks/data-structures/` |

Для каждой задачи можно использовать такой шаблон:

```md
# Название задачи

## Условие

...

## Входные данные

...

## Выходные данные

...

## Пример

```text
Input:
...

Output:
...
```

## Идея решения

...

## Сложность

- Время: `O(...)`
- Память: `O(...)`
```

---

## Полезные обозначения

| Обозначение | Значение |
|---|---|
| `n` | Размер входных данных |
| `T(n)` | Время работы алгоритма |
| `O(...)` | Верхняя оценка сложности |
| `Ω(...)` | Нижняя оценка сложности |
| `Θ(...)` | Точная асимптотическая оценка |
